LUMEN LMS

Learning Management System (LMS) - Техническое задание

Общее описание системы

LMS - система управления обучением, предназначенная для образовательного процесса. Система обеспечивает взаимодействие между студентами и преподавателями через веб-интерфейс и Telegram приложения, с использованием Google Sheets в качестве базы данных.

Функциональные требования

Система аутентификации

- Интеграция с ITMO ID через OAuth 2.0

- Автоматическая регистрация пользователей после успешной аутентификации

- Разделение ролей: студент, лектор, практик, ментор, администратор

Управление курсами

- Автоматическое получение курсов студентов из ИСУ

- Ручная привязка курсов к системе

- Управление расписанием и материалами курсов

Функциональность для студентов

- Просмотр оценок по курсам

- Статус код-ревью с отслеживанием прогресса

- Доступ к работам и учебным материалам

- Система обратной связи с преподавателями

- Автоматическая выдача репозиториев для выполнения заданий

Функциональность для преподавателей

**Admin Panel**

- Панель администратора с полным контролем системы

- Разделение прав доступа по ролям:

- Лектор: управление лекциями, материалами

- Практик: создание и проверка практических работ

- Ментор: код-ревью, консультации студентов

**Управление оценками**

- Выставление баллов за работы

- Комплексная информация по студенту:

- Все оценки за работы

- История комментариев

- Архив код-ревью

**Система работ**

- Конструктор создания работ с шаблонами

- Очередь запросов на код-ревью

- Интеграция с "бан-машиной" для автоматической проверки

- Генерация ростера студентов

**Аудит и мониторинг**

- Логирование всех действий пользователей

- Отслеживание изменений в системе

- Статистика активности студентов и преподавателей

Нефункциональные требования

Производительность

- Время отклика системы: не более 2 секунд для основных операций

- Обработка запросов: поддержка до 1000 одновременных пользователей

- Оптимизация загрузки интерфейса для медленных соединений

Безопасность

- Защита данных пользователей в соответствии с ФЗ152

- Валидация входных данных на всех уровнях системы

- Регулярное резервное копирование данных из Google Sheets

- HTTPS шифрование всех передаваемых данных

Совместимость

- Кросс-браузерная поддержка

- Адаптивный дизайн для мобильных устройств

- Telegram Apps для основных функций системы

Технические требования

- Интеграция с Google Sheets API

- Поддержка OAuth 2.0 для ITMO ID

- Webhook-уведомления для Telegram бота

Надежность

- Доступность системы: 99.5% в рабочее время

- Восстановление после сбоев: не более 30 минут

- Обработка ошибок с информативными сообщениями для пользователей

Масштабируемость

- Архитектура, позволяющая добавлять новые модули

- Поддержка увеличения количества пользователей и курсов

- Гибкая система ролей для расширения функциональности

Технологический стек

- Backend: Node Js

- Frontend: React TS

- База данных: Google Sheets API

- Аутентификация: OAuth 2.0 ITMO ID

- Telegram Bot: LMS

- Хостинг: облачная платформа

- Мониторинг: логирование и базовая аналитика

Интерфейсы интеграции

- ITMO ID OAuth 2.0 - аутентификация

- ИСУ API - получение данных о курсах

- Google Sheets API - хранение данных

- Telegram Bot API - мессенджер интеграция

- GitHub API - управление репозиториями
